# TravelMate 동남아 여행 도우미 챗봇 기획서

## 1. 프로젝트 개요

### 프로젝트명

**TravelMate: Python·Gradio 기반 동남아 여행 도우미 챗봇**

### 프로젝트 목적

본 프로젝트는 **동남아 여행을 준비하는 사용자**가 여행 전이나 여행 중에 필요한 정보를 하나의 챗봇에서 쉽게 확인할 수 있도록 돕는 **여행 도우미 챗봇**을 제작하는 것을 목표로 한다.

사용자는 자연어로 질문을 입력하고, 챗봇은 질문의 의도를 분석하여 적절한 tool을 실행한다. 이를 통해 동남아 여행에 필요한 환율과 경비 계산, 날씨 조회, 여행지 추천, 맛집 및 관광지 추천, 여행 형태별 추천, 번역 기능을 제공한다.

### 개발 형태

- 개발 언어는 **Python**으로 한다.
- 웹 기반 챗봇 형태로 제작한다.
- UI는 **Gradio Blocks**를 사용한다.
- 사용자는 입력창에 질문을 입력하고, 챗봇은 대화형으로 답변한다.
- 초보자도 이해하기 쉽도록 코드를 직관적으로 작성한다. 단, 초기 버전의 UI 코드는 `app.py`에서 한눈에 확인할 수 있도록 구성하고, 환율·날씨·검색·번역 Tool 코드는 기능별 파일로 분리한다.
- 초기 버전은 JSON 샘플 데이터와 기본 함수로 구현하고, 최종 버전은 **외부 Open API**와 연결하여 실제 환율, 날씨, 장소, 번역 정보를 제공한다.
- 최종 결과물은 로컬 실행용 데모가 아니라, 사용자가 접속할 수 있는 **공개 URL 배포**를 목표로 한다.
- 배포 환경에서는 API 키를 코드에 직접 작성하지 않고, 환경 변수 또는 배포 플랫폼의 Secret 설정으로 관리한다.

## 3. 핵심 기능 요약

본 챗봇은 최소 4가지 이상의 tool을 포함한다.

| 구분 | Tool 이름 | 주요 기능 |
|---|---|---|
| 1 | 환율 / 경비 계산 Tool | 환율 변환, 여행 예산 계산, 1인당 경비 계산 |
| 2 | 날씨 조회 Tool | 현재 날씨, 예보, 기온, 강수 확률, 옷차림 추천 |
| 3 | 여행지 검색 및 추천 Tool | 일정, 예산, 테마, 여행 형태 기반 여행지 추천, 맛집 추천, 관광지 추천 |
| 4 | 번역 Tool | 여행 회화 번역, 한국어 ↔ 외국어 번역 |

최종 버전에서는 위 tool들이 단순 계산이나 샘플 데이터에만 의존하지 않고, 외부 Open API를 통해 최신 정보를 가져오도록 확장한다.

## 4. 전체 챗봇 동작 흐름

```text
사용자 입력
   ↓
사용자 질문 의도 분석
   ↓
필요한 Tool 선택
   ↓
Tool 실행
   ↓
결과 정리
   ↓
사용자에게 자연어로 답변
```

### 복합 질문 처리 흐름

사용자가 한 번에 여러 기능을 요청할 수도 있으므로, 하나의 질문에서 여러 tool을 동시에 사용할 수 있도록 설계한다.

예시 입력:

```text
이번 주말 다낭 날씨 보고 3박 4일 커플 맛집 여행 코스 추천해줘.
```

처리 과정:

```text
1. 여행지: 다낭 추출
2. 기간: 이번 주말, 3박 4일 추출
3. 테마: 맛집 여행 추출
4. 여행 형태: 커플여행 추출
5. 날씨 조회 Tool 실행
6. 여행지 검색 및 추천 Tool 실행
7. 날씨와 여행 형태를 고려한 여행 일정 생성
8. 최종 답변 제공
```

## 5. Tool 상세 기획

# 5-1. 환율 / 경비 계산 Tool

## 기능 설명

사용자가 입력한 금액을 다른 통화로 변환하거나, 여행 경비를 계산하는 기능이다.

여행 중 자주 필요한 환율 계산과 예산 계산을 지원한다. 예산 계산은 숙박비, 식비, 교통비, 관광비, 쇼핑비 등을 합산하고, 인원수에 따라 1인당 경비도 계산할 수 있다.

## 주요 기능

| 기능 | 설명 |
|---|---|
| 환율 변환 | 원화, 달러, 엔화, 유로 등 통화 변환 |
| 총 경비 계산 | 숙박비, 식비, 교통비, 관광비 등을 합산 |
| 1인당 경비 계산 | 총 경비를 인원수로 나누어 계산 |
| 예산 적합성 판단 | 입력한 예산이 여행 일정에 적절한지 판단 |
| 항목별 경비 안내 | 항공권, 숙박, 식비, 교통, 관광비 등으로 분류 |

## 입력 예시

```text
100달러는 원화로 얼마야?
```

```text
베트남 여행 3박 4일 경비 계산해줘.
```

```text
숙박비 40만원, 식비 20만원, 교통비 10만원이면 총 얼마야?
```

```text
4명이서 총 120만원이면 1인당 얼마야?
```

## 출력 예시

```text
100 USD는 약 138,000원입니다.
단, 실제 환율은 환전 시점과 수수료에 따라 달라질 수 있습니다.
```

```text
총 예상 경비는 700,000원입니다.
4명이 함께 여행한다면 1인당 약 175,000원입니다.
```

## 필요 데이터

- 금액
- 기준 통화
- 변환 대상 통화
- 여행 기간
- 인원수
- 숙박비
- 식비
- 교통비
- 관광비
- 쇼핑비

# 5-2. 날씨 조회 Tool

## 기능 설명

사용자가 입력한 여행지의 현재 날씨 또는 여행 기간의 날씨 예보를 제공한다.

단순히 날씨 정보만 보여주는 것이 아니라, 여행 상황에 맞게 옷차림과 준비물도 함께 추천한다.

## 주요 기능

| 기능 | 설명 |
|---|---|
| 현재 날씨 조회 | 특정 도시의 현재 날씨 확인 |
| 날짜별 예보 조회 | 내일, 이번 주말, 여행 기간 날씨 확인 |
| 기온 안내 | 최저 / 최고 기온 제공 |
| 강수 확률 안내 | 비나 눈이 올 가능성 안내 |
| 옷차림 추천 | 기온과 날씨에 맞는 옷차림 추천 |
| 준비물 추천 | 우산, 외투, 선크림 등 준비물 추천 |

## 입력 예시

```text
내일 방콕 날씨 어때?
```

```text
이번 주말 다낭 날씨 알려줘.
```

```text
싱가포르 여행 갈 때 옷차림 추천해줘.
```

## 출력 예시

```text
방콕의 내일 예상 기온은 최저 26도, 최고 33도입니다.
오후에 비가 올 가능성이 있으므로 작은 우산을 챙기는 것을 추천합니다.
```

## 필요 데이터

- 날짜
- 현재 날씨
- 최저 기온
- 최고 기온
- 강수 확률
- 습도
- 풍속

# 5-3. 일정·예산·테마·여행 형태 기반 여행지 검색 및 추천 Tool

## 기능 설명

사용자가 입력한 **여행 일정, 예산, 출발지, 동남아 여행 지역, 여행 스타일, 선호 테마, 여행 형태**를 분석하여 조건에 맞는 여행지를 추천한다.

이 tool은 단순히 도시만 추천하지 않는다. 추천 여행지에서 즐길 수 있는 **맛집, 관광지, 카페, 쇼핑 장소, 자연 명소, 역사·문화 장소, 액티비티** 등을 함께 추천한다.

또한 사용자가 원하는 일정에 맞춰 1일차, 2일차, 3일차처럼 여행 코스도 구성할 수 있다. 혼자여행, 가족여행, 커플여행처럼 여행 형태를 선택하면 해당 형태에 맞는 장소, 동선, 숙소 위치, 주의사항도 함께 반영한다.

## 주요 기능

| 기능 | 설명 |
|---|---|
| 일정 기반 여행지 추천 | 1박 2일, 2박 3일, 3박 4일 등 일정에 맞는 여행지 추천 |
| 예산 기반 여행지 추천 | 사용자의 예산에 적합한 여행지 추천 |
| 여행 형태 기반 추천 | 혼자여행, 가족여행, 커플여행 등 여행 형태에 맞는 추천 |
| 지역 기반 추천 | 태국, 베트남, 싱가포르, 말레이시아, 필리핀, 인도네시아 등 동남아 지역 추천 |
| 맛집 추천 | 지역 대표 음식, 인기 맛집, 현지 음식 추천 |
| 관광지 추천 | 유명 명소, 랜드마크, 박물관, 전망대 등 추천 |
| 카페 추천 | 감성 카페, 디저트 카페, 뷰 좋은 카페 추천 |
| 쇼핑 추천 | 전통시장, 쇼핑몰, 기념품 거리 추천 |
| 자연 명소 추천 | 바다, 산, 공원, 호수, 산책 코스 추천 |
| 역사·문화 추천 | 사찰, 궁궐, 박물관, 유적지, 전통 마을 추천 |
| 액티비티 추천 | 놀이공원, 체험 활동, 수상 스포츠 등 추천 |
| 테마별 코스 추천 | 맛집 여행, 힐링 여행, 액티비티 여행 등 맞춤 코스 제공 |
| 일정표 생성 | 여행 기간에 맞춰 일자별 코스 생성 |

## 추천 테마 분류

| 테마 | 추천 내용 예시 |
|---|---|
| 맛집 여행 | 지역 대표 음식, 유명 식당, 현지인 맛집 |
| 관광 여행 | 대표 명소, 랜드마크, 인기 관광지 |
| 힐링 여행 | 조용한 숙소, 자연 명소, 산책 코스 |
| 카페 여행 | 감성 카페, 디저트 카페, 뷰 좋은 카페 |
| 쇼핑 여행 | 백화점, 시장, 기념품 거리, 쇼핑몰 |
| 역사·문화 여행 | 박물관, 유적지, 전통 마을, 사찰 |
| 자연 여행 | 바다, 산, 숲, 호수, 공원 |
| 액티비티 여행 | 놀이공원, 수상 스포츠, 체험 활동 |
| 가족 여행 | 아이와 가기 좋은 장소, 체험형 관광지 |
| 커플 여행 | 야경 명소, 분위기 좋은 식당, 산책 코스 |
| 혼자 여행 | 이동이 편하고 안전한 여행지, 혼밥 가능 장소 |

## 여행 형태 분류

사용자는 여행지 추천을 받을 때 여행 형태를 설정할 수 있다. 챗봇은 선택된 여행 형태에 따라 추천 기준을 다르게 적용한다.

| 여행 형태 | 추천 기준 |
|---|---|
| 혼자여행 | 안전한 지역, 대중교통 접근성, 혼밥 가능 맛집, 이동이 단순한 코스 |
| 가족여행 | 아이 또는 부모님과 함께 가기 좋은 장소, 무리 없는 동선, 휴식 시간이 있는 일정 |
| 커플여행 | 야경 명소, 분위기 좋은 맛집, 사진 찍기 좋은 장소, 여유 있는 산책 코스 |
| 친구와 여행 | 액티비티, 쇼핑, 맛집, 밤에도 즐길 수 있는 번화가 중심 코스 |

## 사용자 입력 조건

| 입력 조건 | 예시 |
|---|---|
| 여행 기간 | 당일치기, 1박 2일, 2박 3일, 3박 4일, 일주일 |
| 예산 | 20만원, 80만원, 150만원 |
| 출발지 | 서울, 부산, 인천공항, 김해공항 |
| 여행 지역 | 태국, 베트남, 싱가포르, 말레이시아, 필리핀, 인도네시아, 특정 도시 |
| 여행 테마 | 맛집, 관광지, 카페, 쇼핑, 자연, 역사, 액티비티 |
| 여행 형태 | 혼자여행, 가족여행, 커플여행, 친구와 여행 |
| 여행 분위기 | 조용한 여행, 활동적인 여행, 여유로운 여행 |
| 이동 방식 | 대중교통, 렌터카, 도보 중심 |

## 입력 예시

```text
3박 4일 일정이고 예산은 80만원이야. 동남아에서 맛집 많은 여행지 추천해줘.
```

```text
다낭 3박 4일 가족여행 갈 건데 관광지랑 맛집 중심으로 코스 짜줘.
```

## 의도 분류 함수 예시

```python
def select_tool(user_input):
    # 환율 또는 경비 계산 관련 질문인지 확인한다
    if "환율" in user_input or "달러" in user_input or "엔화" in user_input:
        return "exchange"

    # 날씨 관련 질문인지 확인한다
    elif "날씨" in user_input or "기온" in user_input or "비" in user_input:
        return "weather"

    # 여행지 추천 관련 질문인지 확인한다
    elif "추천" in user_input or "여행지" in user_input or "코스" in user_input:
        return "travel"

    # 번역 관련 질문인지 확인한다
    elif "번역" in user_input or "영어로" in user_input or "태국어로" in user_input or "베트남어로" in user_input:
        return "translate"

    # 어떤 기능인지 판단하기 어려운 경우
    else:
        return "general"
```

## 7. TravelMate UI/UX 설계

## 7-1. UI/UX 설계 목적

TravelMate는 **귀여운 로봇 캐릭터가 동남아 여행 가이드처럼 친절하게 안내해주는 웹 기반 챗봇**을 목표로 한다.

사용자는 복잡한 여행 사이트를 여러 개 열어보지 않아도, 한 화면에서 여행지 추천, 환율과 경비 계산, 날씨와 옷차림 추천, 여행 회화 번역을 사용할 수 있어야 한다.

UI는 Gradio Blocks를 사용하여 제작하며, PC와 모바일 화면 모두에서 간단하게 사용할 수 있도록 구성한다.

---

## 7-2. UI/UX 핵심 콘셉트

| 항목 | 설계 방향 |
|---|---|
| 프로젝트 이름 | TravelMate |

| 캐릭터 콘셉트 | 귀여운 로봇 캐릭터가 여행 가이드처럼 말해주는 챗봇 |
| 전체 분위기 | 파스텔톤, 밝고 친근한 동남아 여행 분위기 |
| 화면 스타일 | 둥근 카드형 UI, 말풍선 디자인, 부드러운 그림자, 여유 있는 간격 |
| 캐릭터 배치 | 로봇 캐릭터를 화면 왼쪽에 고정 |
| 추천 방식 | 사용자가 여행지나 조건을 입력하면 로봇이 친절하게 추천해주는 느낌 |
| 장식 방향 | 지나치게 화려한 여행 사진보다는 아이콘과 간단한 그래픽 중심 |

## 7-3. 디자인 콘셉트 상세

TravelMate의 디자인은 **밝고 친근한 동남아 여행 도우미** 느낌을 중심으로 한다.

- 초보 여행자도 바로 사용할 수 있는 단순한 화면을 제공한다.
- 여행 앱처럼 깔끔하고 현대적인 디자인을 적용한다.
- 복잡한 메뉴를 줄이고, 빠른 기능 카드와 예시 질문 버튼을 통해 쉽게 사용하게 한다.
- 둥근 카드, 말풍선, 부드러운 그림자, 넉넉한 여백을 사용한다.
- 사용자의 메시지는 오른쪽, 챗봇 메시지는 왼쪽에 배치하여 대화 흐름을 직관적으로 보여준다.
- 로봇 캐릭터는 화면 왼쪽에 고정하여 챗봇의 정체성을 강조한다.

## 7-4. 색상 설계

| 구분 | 색상 방향 | 사용 위치 |
|---|---|---|
| 메인 색상 | 청록색 또는 에메랄드색 | 주요 버튼, 로고, 선택된 옵션, Tool 배지 |
| 보조 색상 | 밝은 베이지색 또는 모래색 | 카드 배경, 섹션 배경, 안내 영역 |
| 강조 색상 | 주황색 또는 코랄색 | 전송 버튼, 중요한 안내, 추천 강조 |
| 배경색 | 흰색 또는 매우 밝은 회색 | 전체 페이지 배경 |
| 글자색 | 검은색에 가까운 진한 회색 | 본문, 제목, 입력 텍스트 |

### 접근성 고려

- 버튼과 글자 사이의 명도 대비를 충분히 확보한다.
- 중요한 값인 금액, 기온, 강수 확률은 **굵게 표시**한다.
- 색상만으로 정보를 구분하지 않고, 아이콘과 텍스트를 함께 제공한다.

```text
┌───────────────────────────────────────────────┐
```

- 모바일 화면에서는 챗봇 상단에 작게 표시하거나 카드 형태로 배치한다.

- 귀여운 로봇 가이드 느낌
- 둥근 얼굴, 작은 안테나, 여행 가방 또는 지도 아이콘과 함께 표현

|---|---|---|
| 환율·경비 계산 | 💱 | 원화, 바트, 동, 달러 등을 환산하고 여행 경비를 계산합니다. |
| 여행지 검색 | 🧭 | 동남아 여행지, 맛집, 관광지, 카페를 추천합니다. |
| 여행 회화 번역 | 💬 | 식당, 호텔, 공항에서 쓸 문장을 번역합니다. |
| 날씨 조회 | ☀️ | 여행지 날씨와 옷차림을 추천합니다. |

### 기능 카드 동작

- 사용자가 카드를 클릭하면 관련 예시 질문이 입력창에 들어간다.
- 카드에는 너무 긴 설명을 넣지 않고, 한 문장으로 이해할 수 있게 작성한다.
- 카드 디자인은 둥근 모서리와 부드러운 그림자를 적용한다.

## 7-9. 여행 조건 입력 영역 설계

사용자가 자연어로 질문하지 않아도 조건을 선택하여 여행지를 추천받을 수 있도록 입력 영역을 제공한다.

### 입력 항목

| 입력 항목 | Gradio 컴포넌트 예시 | 설명 |
|---|---|---|
| 여행 국가 또는 도시 | `gr.Dropdown()` 또는 `gr.Textbox()` | 태국, 베트남, 방콕, 다낭 등 입력 |
| 여행 기간 | `gr.Dropdown()` | 당일치기, 1박 2일, 2박 3일, 3박 4일, 4박 5일 |
| 여행 인원 | `gr.Number()` 또는 `gr.Slider()` | 1명, 2명, 3명 이상 |
| 총예산 | `gr.Number()` 또는 `gr.Slider()` | 원화 기준 총예산 입력 |
| 여행 테마 | `gr.CheckboxGroup()` | 맛집, 관광, 휴양, 쇼핑, 자연, 액티비티 |
| 동행자 유형 | `gr.Radio()` | 혼자, 친구, 커플, 가족 |

### 여행 테마 선택값

```text
맛집 / 관광 / 휴양 / 쇼핑 / 자연 / 액티비티
```

### 동행자 유형 선택값

```text
혼자 / 친구 / 커플 / 가족
```

## 7-10. 챗봇 영역 설계

챗봇 영역은 사용자가 TravelMate와 대화하는 핵심 공간이다.

### 메시지 배치

| 메시지 유형 | 위치 | 디자인 |
|---|---|---|
| 사용자 메시지 | 오른쪽 | 밝은 청록색 계열 말풍선 |
| 챗봇 메시지 | 왼쪽 | 흰색 또는 밝은 베이지색 말풍선 |
| Tool 배지 | 챗봇 답변 상단 | 작은 라벨 형태로 표시 |

### 하단 입력 영역

- 메시지 입력창은 화면 하단에 고정한다.
- 전송 버튼을 명확하게 표시한다.
- 대화 초기화 버튼을 추가한다.
- 모바일 화면에서도 입력창과 버튼이 잘리지 않도록 반응형으로 구성한다.

### Tool 사용 배지 예시

```text
[날씨 Tool 사용]
[환율 Tool 사용]
[검색 Tool 사용]
[번역 Tool 사용]
```

복합 질문의 경우 여러 배지를 함께 표시한다.

```text
[날씨 Tool 사용] [환율 Tool 사용]
```

## 7-14. Gradio Blocks 구현 조건

TravelMate는 Gradio의 `Blocks`를 사용하여 화면을 구성한다.

| 구현 조건 | 설명 |
|---|---|
| Gradio Blocks 사용 | `gr.Blocks()`로 레이아웃을 직접 구성한다. |
| 복잡한 JavaScript 제외 | 별도의 복잡한 JavaScript는 사용하지 않는다. |
| 간단한 CSS만 사용 | 둥근 카드, 색상, 여백, 그림자 정도만 CSS로 조정한다. |
| UI 코드는 app.py에 작성 | 초기 버전에서는 UI 코드를 기능별 파일로 나누지 않고 `app.py`에서 쉽게 확인할 수 있게 한다. |
| 한국어 주석 작성 | 초보자가 수정할 수 있도록 주요 코드에 한국어 주석을 작성한다. |
| 생략 가능한 인자 최소화 | 기본값이거나 생략 가능한 인자는 코드에서 최대한 제외한다. |
| 반응형 고려 | PC와 모바일 화면에서 입력창, 버튼, 카드가 잘리지 않도록 구성한다. |
| URL 배포 고려 | 배포 후 공개 URL에서도 동일한 화면이 안정적으로 보이도록 구성한다. |

### Gradio 레이아웃 예시

아래 예시는 기획서 이해를 돕기 위한 참고용이다. 실제 개발용 `plan.md`에서는 코드 예시 대신 체크리스트 작업 단위로 정리한다.

```python
import gradio as gr

# TravelMate 앱의 기본 화면을 만든다
with gr.Blocks() as app:
    gr.Markdown("# TravelMate")
    gr.Markdown("동남아 여행에 필요한 정보를 한 번에 확인하세요.")

    with gr.Row():
        with gr.Column(scale=1):
            gr.Markdown("🤖 TravelMate 로봇 가이드")
            country_input = gr.Dropdown(
                choices=["태국", "베트남", "필리핀", "인도네시아", "말레이시아", "싱가포르"],
                label="여행 국가 또는 도시"
            )
            period_input = gr.Dropdown(
                choices=["1박 2일", "2박 3일", "3박 4일", "4박 5일"],
                label="여행 기간"
            )
            budget_input = gr.Number(label="총예산")
            theme_input = gr.CheckboxGroup(
                choices=["맛집", "관광", "휴양", "쇼핑", "자연", "액티비티"],
                label="여행 테마"
            )
            companion_input = gr.Radio(
                choices=["혼자", "친구", "커플", "가족"],
                label="동행자 유형"
            )

        with gr.Column(scale=2):
            chatbot = gr.Chatbot(label="TravelMate")
            user_message = gr.Textbox(label="메시지를 입력하세요")
            send_button = gr.Button("전송")
            clear_button = gr.Button("대화 초기화")

    gr.Examples(
        examples=[
            "방콕 이번 주말 날씨 알려줘.",
            "100만 원을 태국 바트로 환산해줘.",
            "다낭 3박 4일 맛집 여행 코스 추천해줘.",
            "계산서 주세요를 베트남어로 번역해줘.",
            "예산 150만 원으로 갈 만한 동남아 여행지 추천해줘."
        ],
        inputs=user_message
    )

app.launch()
```

### 간단한 CSS 적용 방향

```text
- 전체 배경은 밝은 회색 또는 흰색으로 설정한다.
- 카드와 챗봇 말풍선은 둥근 모서리를 적용한다.
- 버튼은 청록색 또는 코랄색 계열로 강조한다.
- 카드에는 약한 그림자를 적용한다.
- 모바일 화면에서는 좌우 영역이 세로로 쌓이게 한다.
```

## 7-15. PC와 모바일 화면 고려사항

| 화면 | 설계 방식 |
|---|---|
| PC 화면 | 왼쪽에 로봇 캐릭터와 여행 조건 입력 영역, 오른쪽에 빠른 기능 카드와 챗봇 영역 배치 |
| 태블릿 화면 | 왼쪽 영역 폭을 줄이고, 챗봇 영역을 넓게 표시 |
| 모바일 화면 | 로봇 캐릭터와 조건 입력 영역을 상단에 배치하고, 챗봇 영역과 입력창을 아래에 배치 |

### 모바일 UX 기준

- 입력창과 전송 버튼이 화면 밖으로 밀리지 않게 한다.
- 기능 카드는 2열 또는 1열로 자동 정렬되게 한다.
- 긴 답변은 접기보다는 제목과 목록으로 나누어 읽기 쉽게 보여준다.
- 예산, 여행 기간, 동행자 선택은 터치하기 쉬운 크기로 만든다.

# 9-8. api_clients 폴더

최종 버전에서 외부 Open API와 직접 통신하는 코드를 모아두는 폴더이다.

`tools` 폴더의 파일은 사용자의 요청을 처리하는 역할을 하고, `api_clients` 폴더의 파일은 실제 API 요청과 응답 처리를 담당한다. 이렇게 분리하면 API가 바뀌어도 전체 챗봇 코드를 크게 수정하지 않고 API 연결 부분만 고칠 수 있다.

| 파일 | 역할 |
|---|---|
| `exchange_api.py` | 환율 API에 요청을 보내고 환율 데이터를 가져온다. |
| `weather_api.py` | 날씨 API에 요청을 보내고 현재 날씨와 예보 데이터를 가져온다. |
| `places_api.py` | 장소 검색 API에 요청을 보내고 맛집, 관광지, 카페 정보를 가져온다. |
| `translation_api.py` | 번역 API에 요청을 보내고 번역 결과를 가져온다. |

## 10-3. 설정 데이터

API 키, 기본 통화, 기본 언어 등의 설정은 `settings.json`에 저장한다.

```json
{
  "default_currency": "KRW",
  "default_language": "ko",
  "use_live_api": true,
  "default_region": "Southeast Asia"
}
```

`settings.json`에는 공개되어도 문제가 없는 기본 설정만 저장한다. API 키는 공개 저장소에 올라갈 수 있는 JSON 파일에 저장하지 않는다. 로컬 개발에서는 `.env` 파일을 사용하고, 최종 URL 배포 환경에서는 배포 플랫폼의 Secret 또는 환경 변수 설정을 사용한다.

### .env.example 예시

```text
EXCHANGE_API_KEY=여기에_환율_API_키_입력
WEATHER_API_KEY=여기에_날씨_API_키_입력
PLACES_API_KEY=여기에_장소검색_API_키_입력
TRANSLATION_API_KEY=여기에_번역_API_키_입력
```

`.env.example`은 입력해야 할 환경 변수 이름만 안내하는 파일이다. 실제 API 키가 들어 있는 `.env` 파일은 공개 저장소에 올리지 않는다.

## 11. 비개발자 친화적 코드 구성 조건

## 목적

본 프로젝트는 개발 경험이 많지 않은 사람도 코드를 읽고 이해할 수 있도록 **쉽고 직관적인 코드 구조**로 작성한다.

코드는 단순히 동작하는 것뿐만 아니라, 이후 사용자가 직접 여행지 데이터, 예시 질문, 추천 기준 등을 수정할 수 있도록 구성한다.

## 코드 작성 원칙

| 원칙 | 설명 |
|---|---|
| 쉬운 변수명 사용 | 변수명만 봐도 의미를 알 수 있도록 작성한다. |
| 기능별 파일 분리 | 환율, 날씨, 여행지 추천, 번역 Tool은 각각 별도 파일로 분리한다. 단, 초기 버전의 UI 코드는 `app.py`에서 한눈에 확인할 수 있게 작성한다. |
| 짧은 함수 작성 | 하나의 함수가 너무 많은 일을 하지 않도록 한다. |
| 한국어 주석 작성 | 코드의 주요 부분마다 설명 주석을 추가한다. |
| 설정값 분리 | 여행지 데이터, 예시 질문, API 키 등은 별도 파일로 관리한다. |
| 복잡한 문법 최소화 | 초보자가 이해하기 어려운 고급 문법 사용을 줄인다. |
| 친절한 오류 메시지 | 오류 발생 시 사용자가 이해할 수 있는 문장으로 안내한다. |
| README 상세 작성 | 실행 방법과 데이터 수정 방법을 문서로 제공한다. |

## 코드 작성 방식

### 좋지 않은 예시

```python
def calc(x, y):
    return x / y
```

위 코드는 `x`, `y`가 무엇을 의미하는지 알기 어렵다.

### 개선된 예시

```python
def calculate_cost_per_person(total_cost, people_count):
    """
    전체 여행 경비를 인원수로 나누어 1인당 경비를 계산한다.
    """
    cost_per_person = total_cost / people_count
    return cost_per_person
```

위 코드는 함수명과 변수명만 봐도 어떤 기능인지 쉽게 알 수 있다.

## 비개발자가 수정하기 쉬운 구조

수정이 자주 필요한 정보는 Python 코드 안에 직접 작성하지 않는다.

예를 들어 여행지, 맛집, 관광지는 `travel_spots.json` 파일에 저장한다. 사용자는 이 파일만 수정해도 새로운 여행지나 추천 장소를 추가할 수 있다.

```json
{
  "나트랑": {
    "country": "베트남",
    "recommended_days": "3박 4일",
    "budget_range": "80만~140만원",
    "themes": {
      "자연": ["나트랑 해변", "혼문섬", "포나가르 사원"],
```

## 13. 예외 처리

| 상황 | 처리 방식 |
|---|---|
| 도시명을 알 수 없음 | “어느 도시의 정보를 원하시나요?”라고 안내 |
| 날짜가 없음 | 현재 날짜 또는 기본 예보 기준으로 조회 |
| 예산이 없음 | 일반적인 평균 예산 기준으로 추천 |
| 여행 기간이 없음 | 2박 3일 또는 기본 추천 일정으로 안내 |
| 테마가 없음 | 관광지, 맛집, 카페를 골고루 포함한 기본 코스 추천 |
| 여행 형태가 없음 | 일반여행 기준으로 추천하고, 필요 시 혼자여행/가족여행/커플여행 선택을 안내 |
| 환율 정보 조회 실패 | “현재 환율 정보를 가져오지 못했습니다”라고 안내 |
| 번역 언어가 없음 | 기본값을 영어로 설정하거나 언어 선택 요청 |
| API 오류 발생 | 사용자가 이해할 수 있는 친절한 오류 메시지 제공 |
| 질문 의도 파악 실패 | 사용 가능한 기능 목록 안내 |

## 14. Open API 연동 계획

최종적인 형태는 외부 Open API와 연결하여 최신 정보를 제공하는 챗봇으로 구현한다.

초기 개발 단계에서는 API 없이 샘플 데이터와 기본 함수로 동작하도록 만들고, 이후 Open API를 연결하여 실제 환율, 날씨, 장소, 번역 정보를 가져오는 방식으로 확장한다.

## 14-1. Open API 연동 대상

| 기능 | 연동할 Open API 유형 | 사용 목적 |
|---|---|---|
| 환율 / 경비 계산 | 환율 API | 실시간 환율을 가져와 원화, 달러, 바트, 동, 싱가포르 달러 등으로 변환 |
| 날씨 조회 | 날씨 API | 동남아 주요 도시의 현재 날씨, 예보, 기온, 강수 확률 조회 |
| 여행지 검색 및 추천 | 장소 검색 API | 맛집, 관광지, 카페, 쇼핑 장소 등 실제 장소 정보 검색 |
| 번역 | 번역 API | 한국어 문장을 영어, 태국어, 베트남어 등으로 번역 |

## 14-2. 사용 가능한 API 예시

| 기능 | 사용 가능 API 예시 |
|---|---|
| 환율 | ExchangeRate API, Frankfurter API |
| 날씨 | OpenWeatherMap API, WeatherAPI |
| 여행지 검색 | Google Places API, Naver Search API, Kakao Local API |
| 번역 | Papago API, Google Translate API, DeepL API |

## 14-3. Open API 연동 구조

```text
사용자 입력
   ↓
Gradio app.py
   ↓
의도 분석 intent_classifier.py
   ↓
선택된 Tool 실행
   ↓
api_clients 폴더의 API 요청 함수 호출
   ↓
외부 Open API에서 데이터 수신
   ↓
응답 데이터 정리
   ↓
response_formatter.py에서 답변 형식 구성
   ↓
Gradio 챗봇 화면에 출력
```

## 14-4. Open API 연동 시 고려사항

| 고려사항 | 설명 |
|---|---|
| API 키 관리 | API 키는 코드에 직접 작성하지 않고 `.env` 파일, 환경 변수, `settings.json` 등으로 분리한다. |
| 요청 실패 처리 | 네트워크 오류, API 제한, 잘못된 도시명 등의 상황에 대비한다. |
| 기본 데이터 유지 | API 호출에 실패했을 때를 대비해 `travel_spots.json`의 기본 추천 데이터를 유지한다. |
| 응답 형식 통일 | API마다 응답 구조가 다르므로 챗봇이 사용할 수 있는 공통 형식으로 변환한다. |
| 속도 개선 | API 응답이 느릴 경우 로딩 메시지나 간단한 안내 문구를 제공한다. |
| 비용 관리 | 유료 API를 사용할 경우 호출 횟수와 사용량 제한을 고려한다. |

## 14-5. API 실패 시 대체 동작

Open API 연결이 실패해도 챗봇이 완전히 멈추지 않도록 다음과 같이 처리한다.

| 실패 상황 | 대체 처리 |
|---|---|
| 환율 API 오류 | 마지막으로 저장된 환율 또는 안내 메시지 제공 |
| 날씨 API 오류 | “현재 날씨 정보를 가져오지 못했습니다”라고 안내 |
| 장소 검색 API 오류 | `travel_spots.json`의 기본 맛집·관광지 데이터 사용 |
| 번역 API 오류 | “현재 번역 기능을 사용할 수 없습니다”라고 안내 |
| API 키 누락 | README의 API 키 입력 방법을 확인하라고 안내 |

## 15-2. 최종 URL 배포 목표

TravelMate의 최종 결과물은 단순한 로컬 테스트 프로그램이 아니라, 다른 사용자가 접속할 수 있는 **공개 웹 URL**을 가진 챗봇이다.

| 단계 | 목적 | 설명 |
|---|---|---|
| 로컬 실행 | 기능 개발과 테스트 | 개발자가 PC에서 `app.py`를 실행하여 화면과 Tool 동작을 확인한다. |
| 임시 공유 URL | 중간 발표 또는 빠른 시연 | Gradio의 공유 링크 기능을 사용하여 일시적으로 다른 사람에게 테스트 링크를 전달할 수 있다. |
| 최종 배포 URL | 최종 결과물 제출 및 실제 사용 | Hugging Face Spaces, Render, Railway, Cloud Run 등 Python·Gradio 앱을 실행할 수 있는 플랫폼에 배포한다. |

본 프로젝트에서는 Gradio와의 호환성과 구현 난이도를 고려하여 **Hugging Face Spaces 배포를 우선 고려**한다. 단, 수업 또는 프로젝트 환경에서 지정한 배포 플랫폼이 있다면 해당 플랫폼의 요구사항을 따른다.

## 15-3. URL 배포 시 고려사항

| 고려사항 | 설명 |
|---|---|
| 진입 파일 | 배포 플랫폼에서 실행할 메인 파일은 `app.py`로 통일한다. |

| 오류 안내 | 배포 환경에서 API 오류가 발생해도 기술적 메시지 대신 친절한 안내 문장을 출력한다. |
| 사용량 제한 | 무료 API 또는 무료 배포 환경의 호출 횟수, 실행 시간, 트래픽 제한을 고려한다. |

## 15-4. 개발 제약 사항 반영

업로드된 개발 제약 문서를 기준으로, TravelMate 개발은 **기획 모드**와 **실행 모드**를 분리하여 진행한다.

### 기획 모드 제약

- 기획 단계에서는 바로 코드를 작성하지 않는다.
- 요구사항이 불명확하면 한 번에 하나의 질문만 던져 확인한다.
- 요구사항이 충분히 정리되면 사용자의 명시적 승인 후 마크다운 체크리스트 형태의 계획을 작성한다.
- 개발자가 바로 실행할 수 있도록 체크리스트는 한 항목당 하나의 구체적인 작업만 포함한다.
- 실제 개발용 `plan.md`에는 불필요한 코드 예시를 넣지 않고, 작업 단위와 제약 조건을 중심으로 작성한다.

### 실행 모드 제약

- `plan.md`에서 가장 먼저 발견되는 미완료 체크박스 하나만 처리한다.
- 한 번의 실행에서는 하나의 원자적 작업만 수행한다.
- 다음 단계의 작업을 미리 예상해서 함께 처리하지 않는다.
- 새 함수, 클래스, 상수 등을 추가한 경우 다음 체크리스트에서 명시하기 전까지 다른 곳에서 사용하지 않는다.
- 작업 범위 밖의 파일이나 코드는 수정하지 않는다.
- 작업이 끝나면 테스트와 린트 확인 후 해당 작업만 완료 처리한다.
- 작업 완료 후 발견한 일반적인 주의사항은 `plan.md`의 `Notes` 바로 아래 `Rules & Tips` 섹션에 정리한다.
- 작업 하나를 마치면 다음 작업으로 넘어가지 않고 멈춘다.

## 15-5. Python 프로젝트에 맞춘 검증 기준

| 배포 확인 | 최종 배포 URL에서 챗봇 접속과 Tool 실행이 가능한지 확인한다. |

## 19. 기대 효과

- 여행자가 필요한 정보를 한 곳에서 빠르게 확인할 수 있다.
- 여러 사이트를 따로 검색하지 않아도 된다.
- 환율, 날씨, 번역, 추천 기능을 하나의 챗봇에서 사용할 수 있다.
- 사용자의 일정과 예산에 맞는 여행지를 추천할 수 있다.
- 맛집, 관광지, 카페, 쇼핑 등 테마별 추천이 가능하다.
- 혼자여행, 가족여행, 커플여행 등 여행 형태에 맞는 추천이 가능하다.
- Python과 Gradio를 사용하므로 웹 형태의 챗봇 프로토타입을 빠르게 만들 수 있다.
- 최종 URL 배포를 통해 사용자가 별도 설치 없이 브라우저에서 챗봇을 사용할 수 있다.
- 비개발자도 JSON 파일을 수정하여 여행지 데이터를 쉽게 추가할 수 있다.
- Open API를 연결하여 최신 환율, 날씨, 장소, 번역 정보를 반영할 수 있다.
- 샘플 데이터 기반 프로토타입에서 실제 서비스 형태로 확장할 수 있다.

## 20. 최종 목표

본 프로젝트의 최종 목표는 **여행 중 필요한 정보를 통합 제공하는 Python·Gradio 기반 멀티툴 여행 챗봇**을 구현하고, 사용자가 접속할 수 있는 **최종 웹 URL**로 배포하는 것이다.

챗봇은 사용자의 자연어 질문을 이해하고, 다음 기능을 중심으로 동작한다.

```text
1. 환율 / 경비 계산
2. 날씨 조회
3. 일정·예산·테마·여행 형태 기반 여행지 검색 및 추천
4. 맛집, 관광지, 카페, 쇼핑 장소 추천
5. 여행 회화 번역
```

초기 버전은 키워드 기반 tool 선택 방식과 샘플 JSON 데이터로 구현한다. 최종 버전은 외부 Open API와 연결하여 실시간 환율, 날씨, 장소 검색, 번역 정보를 제공하고, 공개 URL을 통해 접속 가능한 여행 도우미 챗봇으로 확장한다.

UI/UX 측면에서는 귀여운 로봇 캐릭터가 동남아 여행 가이드처럼 안내하는 콘셉트를 적용한다. 전체 화면은 파스텔톤, 둥근 카드형 UI, 말풍선 디자인으로 구성하고, Gradio Blocks를 사용하여 PC와 모바일에서 모두 사용하기 쉬운 웹 챗봇으로 구현한다.

또한 본 프로젝트는 비개발자도 쉽게 이해하고 수정할 수 있도록 다음 조건을 지킨다.

```text
- 기능별 파일을 분리한다.
- 변수명과 함수명은 역할이 명확하게 작성한다.
- 여행지, 맛집, 관광지, 예시 질문 등의 수정 가능한 정보는 별도 JSON 파일로 관리한다.
- 주요 코드에는 한국어 주석을 추가한다.
- Open API 연결 코드는 `api_clients` 폴더에 따로 분리한다.
- API 키는 환경 변수 또는 배포 플랫폼 Secret으로 관리한다.
- 개발은 `plan.md` 체크리스트를 기준으로 한 번에 하나의 작업만 처리한다.
- README 파일에는 실행 방법, 데이터 수정 방법, API 키 입력 방법, 최종 URL 배포 방법을 자세히 작성한다.
```
